# 订单管理系统 Demo - 技术方案设计文档

## 1. 项目概述

### 1.1 项目背景
- **项目名称**: 订单管理系统 Demo
- **版本**: v3.0
- **目标**: 实现轻量级订单管理基础功能，解决"订单的增删改查"核心业务场景
- **定位**: 快速创建、查看、修改、删除订单，以最直观的方式管理订单数据

### 1.2 目标用户
1. **业务操作员**: 日常录入和跟进订单，关注操作效率
2. **小型商家/个体经营者**: 订单量不大，需要简单工具替代 Excel

### 1.3 功能范围
#### ✅ 本期包含功能
- **订单头管理**: 创建/列表/详情/编辑/删除
- **订单头管理**: 筛选搜索、状态变更
- **订单明细管理**: 增删明细行、自动计算合计

#### ❌ 本期不包含功能
- 用户权限、批量导入导出、通知推送
- 支付对接、库存管理、数据分析、多租户

### 3.2 状态流转规则
```
待处理(pending) → 处理中(processing) → 已完成(completed)
    ↓                   ↓
  已取消(cancelled)   已取消(cancelled)
```
- **终态规则**: 已完成(completed)/已取消(cancelled)状态不可逆
- **状态变更**: 仅允许按规则流转，不允许跳转

## 5. API接口设计

### 5.1 接口规范
- **基础URL**: http://localhost:8080/api
- **响应格式**: JSON
- **状态码**: 遵循HTTP标准状态码

### 5.2 订单管理接口

#### 1. 创建订单
- **URL**: POST `/api/orders`
- **请求体**: 
```json
{
  "customerName": "张三",
  "phone": "13800138000",
  "address": "北京市朝阳区",
  "remark": "请尽快发货",
  "orderItems": [
    {
      "productName": "商品A",
      "quantity": 2,
      "unitPrice": 100.00
    }
  ]
}
```

#### 2. 查询订单列表
- **URL**: GET `/api/orders`
- **查询参数**:
  - `pageNo`: 页码，默认1
  - `pageSize`: 每页大小，默认10
  - `orderNo`: 订单编号（模糊查询）
  - `customerName`: 客户姓名（模糊查询）
  - `status`: 订单状态
  - `startTime`: 开始时间
  - `endTime`: 结束时间

#### 3. 获取订单详情
- **URL**: GET `/api/orders/{id}`

#### 4. 更新订单
- **URL**: PUT `/api/orders/{id}`
- **请求体**: 同创建订单，包含所有字段

#### 5. 删除订单
- **URL**: DELETE `/api/orders/{id}`

#### 6. 更新订单状态
- **URL**: PATCH `/api/orders/{id}/status`
- **请求体**:
```json
{
  "status": "processing"
}
```

## 7. 关键业务逻辑

### 7.1 订单号生成规则
```java
public static String generateOrderNo() {
    // 格式: ORD-yyyyMMdd-xxxx
    // 示例: ORD-20240427-0001
    String date = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyyMMdd"));
    // 实际实现包含序列号生成逻辑
    return String.format("ORD-%s-%04d", date, sequence);
}
```

### 7.2 金额计算逻辑
1. **明细小计**: `subtotal = quantity × unitPrice`
2. **商品总数**: `totalItems = Σ(orderItems.quantity)`
3. **总金额**: `totalAmount = Σ(orderItems.subtotal)`

### 7.3 状态流转验证
```java
public boolean canChangeStatus(String currentStatus, String newStatus) {
    // 状态流转规则验证
    Map<String, List<String>> allowedTransitions = Map.of(
        "pending", Arrays.asList("processing", "cancelled"),
        "processing", Arrays.asList("completed", "cancelled"),
        "completed", Collections.emptyList(), // 终态不可变
        "cancelled", Collections.emptyList()  // 终态不可变
    );
    return allowedTransitions.getOrDefault(currentStatus, Collections.emptyList())
            .contains(newStatus);
}
```

-- 创建数据库
CREATE DATABASE order_demo CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'order_user'@'%' IDENTIFIED BY 'OrderDemo@2026';

CREATE TABLE order_header (

    phone VARCHAR(20),

    INDEX idx_status (status),

-- 创建订单明细表

    unit_price DECIMAL(15,2) NOT NULL DEFAULT 0.00,
    subtotal DECIMAL(15,2) NOT NULL DEFAULT 0.00,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES order_header(id) ON DELETE CASCADE,
    INDEX idx_order_id (order_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
