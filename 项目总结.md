## 📊 核心业务逻辑

### **订单状态管理**
- **状态定义**：pending → processing → completed / cancelled
- **状态流转规则**：
  - pending → processing（开始处理）
  - pending → cancelled（取消订单）
  - processing → completed（完成订单）
  - processing → cancelled（取消处理中的订单）
- **终态不可逆**：completed 和 cancelled 状态不能更改

### **订单号生成**
- **格式**：`ORD-yyyyMMdd-xxxx`
- **规则**：每天从0001开始，自动递增
- **线程安全**：使用 AtomicInteger 和 synchronized

### **自动计算**
- **商品小计**：数量 × 单价
- **订单总额**：所有商品小计之和
- **商品总数**：所有商品数量之和

### **软删除机制**
- 使用 `deleted` 字段标记删除
- 删除时设置 `delete_time`
- 查询时自动过滤已删除记录

```sql
CREATE DATABASE order_management_demo 
    CHARACTER SET utf8mb4
```

